Slide 2: PROBLEM DOMAIN & MOTIVATIONS
================================================================================
**The Problem:**
*   **Lack of Security:** Standard raw sockets send data in plaintext (vulnerable to sniffing).
*   **Head-of-Line Blocking:** In basic implementations, sending a large file (e.g., 1GB) blocks all chat messages until the file finishes.
*   **Unreliability:** Network glitches often cause permanent failure for file transfers (no resume capability).

**Our Motivation:**
*   To build a system that is **Secure by Design** using modern cryptography (End-to-End Encryption).
*   To solve traffic congestion by implementing **Priority Queuing** (Chat > Files).
*   To implement practical **Reliability** features like "Resume Download" for large files.

--------------------------------------------------------------------------------

Slide 3: OBJECTIVES / AIMS
================================================================================
1.  **Confidentiality & Integrity:** Ensure no third party can read messages (RSA + AES-GCM).
2.  **Responsiveness (Traffic Shaping):** Ensure chat messages are delivered instantly, even during heavy file uploads.
3.  **Resilience:** Implement robust error recovery (File Resume) to handle network interruptions.
4.  **Usability:** Provide a premium, modern JavaFX GUI with real-time feedback.

Slide 4: PROJECT FEATURES
================================================================================
*   **Secure Authentication:** RSA-2048 Handshake & AES-256 Session Keys.
*   **Real-Time Messaging:** Direct Messages (DM) & Group Chats with instant delivery.
*   **Resumable File Transfer:**
    *   Files are split into small chunks.
    *   Automatic recovery from the last successful chunk after disconnection.
*   **Priority Traffic Shaping:** Text messages "jump the queue" ahead of file chunks.
*   **Group Management:** Dynamic creation and joining of groups.
*   **Universal Chunking:** Uniform processing for improved flow control.

Slide 7: APPLIED NETWORKING CONCEPTS (BACKEND)
================================================================================

We applied these core networking concepts to ensure robustness and efficiency:

1.  **Application-Layer Flow Control (Universal Chunking):**
    *   *Concept:* Breaking large data streams into fixed-size chunks (e.g., 64KB).
    *   *Application:* Prevents channel monopolization. Allows the server to interleave chat messages between file chunks.

2.  **Traffic Shaping / Congestion Control (Priority Queuing):**
    *   *Concept:* Prioritizing time-sensitive data over bulk data.
    *   *Application:* Implemented a `PriorityBlockingQueue` on the server.
        *   **Priority 1:** Chat/Control Packets (Processed immediately).
        *   **Priority 3:** File Chunks (Processed when line is free).

3.  **Reliable Data Transfer (ARQ - Automatic Repeat Request):**
    *   *Concept:* Acknowledgements and Sequence Numbers.
    *   *Application:* Receivers send `ACK` packets for saved chunks. Sender tracks `chunkIndex` to resume exactly where it left off (Selective Repeat logic).

4.  **Security (Hybrid Encryption):**
    *   *Concept:* Combining Asymmetric (RSA) and Symmetric (AES) schemas.
    *   *Application:* RSA for Key Exchange (slow, secure) + AES-GCM for Data Transport (fast, secure, authenticated).

Slide 9: LIMITATIONS AND FUTURE PLAN
================================================================================
**Limitations:**
*   **In-Memory State:** Server restarts cause loss of active group/user lists (data is transient).
*   **No Offline Storage:** Messages sent to offline users are currently not stored for later delivery.

**Future Plans:**
*   **Database Integration:** Implement PostgreSQL to persist user credentials and chat history.
*   **VoIP Support:** Add UDP-based packet handling for Real-Time Voice/Video calls.
*   **Mobile App:** Develop an Android client for cross-platform availability.
